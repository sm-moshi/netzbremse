PostgreSQL-backed Netzbremse services for `m0sh1.cc`.

`netzbremse.de` is a campaign site and measurement project centred on Deutsche
Telekom throttling and alleged net-neutrality violations. The speed test is one
part of that project, used to collect and present measurement data.

This repository contains the application code, container builds, and CI/release
workflow for the two service components used by the infra wrapper chart:

- `measurement`: runs the Netzbremse browser flow and stores results in PostgreSQL
- `dashboard`: serves the web UI and API from PostgreSQL-backed data

## Repository layout

- `cmd/measurement/` — measurement worker entrypoint
- `cmd/dashboard/` — dashboard server entrypoint and embedded static assets
- `internal/` — shared application code
- `db/migrations/` — SQL schema files
- `scripts/netzbremse-browser.mjs` — Puppeteer-based browser runner
- `Dockerfile.measurement` — measurement image build
- `Dockerfile.dashboard` — dashboard image build
- `.woodpecker/` — CI and release pipelines

## Runtime configuration

Database connection:

- `NETZBREMSE_DATABASE_URL`
- or `DATABASE_URL`

Measurement-specific settings:

- `NETZBREMSE_MEASUREMENT_INTERVAL`
- `NETZBREMSE_ENDPOINT`
- `NETZBREMSE_SPEEDTEST_COMMAND`
- `NETZBREMSE_SPEEDTEST_TIMEOUT`
- `NETZBREMSE_IMPORT_DIR`

Dashboard-specific settings:

- `NETZBREMSE_DASHBOARD_LIMIT`

The default measurement command is `node /app/netzbremse-browser.mjs`.

## CI and release flow

- Pull requests and pushes to `main` run Go tests and browser script syntax checks.
- Pushes to `main` also run `semantic-release` against `https://github.com/sm-moshi/netzbremse`.
- Tag pipelines (`v*`) currently build and publish:
  - `ghcr.io/sm-moshi/netzbremse-measurement`
  - `ghcr.io/sm-moshi/netzbremse-dashboard`
- GHCR releases are scanned with Trivy, signed with Cosign, and receive:
  - SPDX SBOM attestations generated by Syft
  - vulnerability attestations generated from Trivy
- Tag pipelines then copy the exact published image digests to:
  - `harbor.m0sh1.cc/apps/netzbremse-measurement`
  - `harbor.m0sh1.cc/apps/netzbremse-dashboard`
- Harbor copies are then signed again for Harbor-native consumers with:
  - Cosign signatures
  - SPDX SBOM attestations
  - vulnerability attestations
  - Notation signatures

Required Woodpecker secrets:

- `github_username`
- `github_token`

- `dhi_token`
- `harbor_username`
- `harbor_token`

- `cosign_password`
